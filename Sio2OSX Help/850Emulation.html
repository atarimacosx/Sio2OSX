Sio2OSX fully emulates the 4 serial ports of the Atari 850 Interface.  Users are allowed to chose to connect each of the 4 available ports of the 850 to a Internet Modem simulation or to additional serial ports connected to the Macintosh.  Connecting to the Internet Modem simulation will allow connection to or hosting of telnet based BBS's.  Connecting to additional serial ports will allow you to connect the Atari to external serial devices.

The status of the 850 Emulation is controled by the 850 power button on the main screen of the program:



By default the 850 Emulation comes up as off.  To turn it on press the On button.  The image above shows the emulation on (note the red light on the 850), and pressing the Off button will turn it off.  Cycling the power on the 850 will reset it's internal state, just like the real device.

850 Port Connection:

Devices, including the Internet Modem, are connected/disconnect from the 850 port by using the 850 Tab in the Preference panel.  Note, when connecting to actual serial ports, the emulation overcomes one limitation of the 850.  On ports 2,3, and 4, not all of the hardware handshake lines were present in the real 850, but the emulation makes no distinction between the 4 ports, and all of the lines are available for use on all ports.

Internet Modem Configuration and Use:

The internet modem simulation in Sio2OSX simulates a Hayes compatible modem connected to one of the 850 ports.  However, this modem "dials" and "answers" calls using network socket connections instead of phone lines, so that the user can connect to Telnet based BBS's or host a Telnet BBS with their Atari.  The internet modem configuration is controlled from the Modem Tab in the Preference panel.

There are two components to the simulation, the command processing and the server.  The command processor is used for both the server and the client.  The server may be enabled or disabled in the Modem Tab in the Preference panel, so if you do not want to have a TCP port open on your system for the server, leave this option disabled.  Once started, the server can be stopped and started through the modem command processor described in the next section.

Internet Modem Commands

The internet modem will respond to a subset of the Hayes AT command set.  The function and syntax of each of the commands it will respond to is shown below.  One restriction that should be noted, is that unlike actual modems, the simulation requires that each command be on a seperate line.

ATA
    This command causes the modem to answer if it has received a RING. (with the internet modem, this means that a client has tried to connect to the server listening on a specific port).  This means the modem will leave command mode, and enter data mode..  If modem Auto Answer is turned on in the configuration, this command is not required, and the modem will automatically go to data mode.

ATD, ATDI, or ATDT
     All three of these commands will cause the modem to "dial" to an internet address (and optionally port number).  If the internet connection is made successfully, the modem will return a CONNECT buad message.  If the connection is not made, the modem will return NO CARRIER.
     Here are some examples.  To dial the bbs ataribbs.com on the default port (23) you would enter:

    ATD ATARIBBS.COM

    To dial the bbs ataribbs.org on port 8888 you would enter:

    ATDT ATARIBBS.ORG 8888

ATDP
     This command is used to dial an internet address and port that were previously stored using the AT&Z command described later, or using the Modem Tab in the Preference panel.
     For example, to dial the first stored host you would enter:

    ATDP 1

ATE
     This command is used to turn the echo of characters from the modem on or off.  With Echo on, the modem will echo all characters it receives from the computer back to the computer, so one can see what one types.  The default for Echo is on, but this default can be changed using the Modem Tab in the Preference panel.  To turn Echo on, you would enter ATE1, to turn it off ATE0.

ATH
     This command is used to hang up an active connection.  It would require the modem be in command mode, which means the escape sequence (by default '+++') would have to have been previously entered to take the modem from data to command mode.

ATI
     This command displays a help summary of the commands the modem accepts.     

ATK
     This command is used to stop (kill) the server portion of the internet simulator.  By default, the server is not enabled, but it may be enabled by using the Modem Tab in the Preference panel.  

ATO   
     This command is used to resume an active connection that has been interrupted using the escape sequence, returning the modem to data mode.

ATR
     This command is used to restart the server portion of the internet simulator, if it has been stopped by the ATK command.

ATS
     This command is used to set the registers in the modem.  Currently, only two options are settable this way.  The format of the command is:

    ATSn=r
     Where n is the register number, and r is the value to store there.  The following are the register numbers and possible values:

    Register 0: Auto Answer
        0 = Off
        1 = On
        Default is Off (unless changed in the  Modem Tab in the Preference panel.)

    Register 2: Escape Charcter
        0 thru 127 = ASCII value of the escape character
        128 thru 255 = Escape character is disabled
        Defualt is 43, which is ASCII for  the '+' character. (unless changed in Modem Tab in the Preference panel.)

ATZ    : Reset Modem
     This command resets the modem, hanging up if there is an active connection.  It sets the Auto Answer, Echo, and Esacpe Character to the defaults which are set in the  Modem Tab in the Preference panel.

AT&F  
     This command resets the modem to the factory defaults. It sets the Auto Answer to Off, Echo to On, and Esacpe Character to '+'.  Note is does not set them to the defaults which are set in the  Modem Tab in the Preference panel.

AT&V  
     This command displays the internet address and ports which have been stored in the modem for later dialing by ATDP.  The numbers may also be edited in the Modem Tab in the Preference panel.

AT&Z
     This command is used to store internet addresses and ports for later use in dialing with the ATDP command.  The format of the command is:

    AT&Zn=address [port]
     Where n is the slot to store the address/port in (1-20), address is the internet address, and optionally port is the port number.  If port is not specified, 23 will be used by default.
     For Example:

    AT&Z1=ataribbs.com 1234
     Will set the first stored address to ataribbs.com with a port number of 1234.

+++   
     When in data mode, typing the esacpe charcter 3 times in a row will force the modem back to command mode.  The character used for escape can be changed using the ATS2= command, or in the Modem Tab in the Preference panel.
